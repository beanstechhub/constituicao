Como funciona

O Fraud Shield avalia cada transação em tempo real usando sliding windows (janelas deslizantes) que rastreiam frequência e volume por usuário. Não depende de IA nem de APIs externas — é 100% determinístico e explicável.

Transação → VelocityCounter → 12 Regras → Scoring → Decisão ↓ ↑ Sliding Windows ML Score (opcional) (in-memory) via interface Scorer

O que é uma Sliding Window?

Uma janela temporal que "desliza" com o tempo. Exemplo: "quantas transações este usuário fez na última 1 hora?" A janela sempre olha para trás a partir do momento atual, descartando dados antigos automaticamente.

Decisões

Cada transação recebe uma de três decisões:

  • Allow — transação normal, sem indicadores de fraude
  • Review — padrão suspeito, requer análise humana antes de liberar
  • Block — padrão fortemente indicativo de fraude, transação retida

Scoring

O score final combina as regras disparadas com um peso opcional de modelo ML:

score = rule_score × (1 - ml_weight) + ml_score × 10 × ml_weight Decisão: score ≥ 8.0 ou risk = critical → Block score ≥ 4.0 ou risk = high → Review caso contrário → Allow

Sem modelo ML configurado (ml_weight = 0), o sistema opera 100% baseado em regras — determinístico e auditável.

Integração

O Fraud Shield é uma biblioteca Go pura (zero dependências externas). Pode ser integrado:

  • Via API REST — endpoint /v1/fraude/avaliar no gateway
  • Como biblioteca Go — importar github.com/beanstechhub/constituicao/fraud-shield
  • Extensível — interfaces Scorer (ML), Persister (banco), Rule (regras customizadas)

Fundamento Constitucional

Art. 170, CF/88 — A ordem econômica, fundada na valorização do trabalho humano e na livre iniciativa, tem por fim assegurar a todos existência digna. A proteção contra fraude financeira é condição para uma economia saudável.

Alinhado à Circular BACEN 3.978/2020 (prevenção à lavagem) e padrões COAF de reporte.
